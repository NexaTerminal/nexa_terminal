## 0. ОПСЕГ

Со ЕДЕН пакет се прават ЕДНА или ПОВЕЌЕ промени истовремено. Корисникот ги одбира промените
(checkbox), внесува податоци по промена (чекор „Полиња по промена" — секоја промена е посебен под-чекор
со **Тековно → Ново** приказ), а системот динамички го составува пакетот документи.

**НИКОГАШ не се менуваат** (само идентификациски, се појавуваат во преамбулите):
- ЕДБ (даночен број, 13 цифри)
- ЕМБС (матичен број, 7 цифри)
- Приоритетна дејност (шифра)

## 2. ДОКУМЕНТИ ВО ПАКЕТОТ

### 2.1 Секогаш (за било која промена) [code: `assembleChildren`]
1. По една **Одлука** за секоја одбрана промена M1–M4, M6, M7
2. **Изјава по член 32** (од управителот) — со авто-генерирана листа на дејствија (види §5)
3. **Изјава за потписи** (чл. 7 ст. 1 ал. 4 од Правилникот) — по едно за СЕКОЈ потписник
4. **Полномошно** за регистрациониот агент — по едно за СЕКОЈ потписник

### 2.2 Условно [code: `assembleChildren`, `flags`]
| Документ | Услов | [code] |
|---|---|---|
| Одлука за измена на Актот (чл. 252 и 253) | барем една M1–M6 или M5 (**НЕ** за само-M7 пакет) | `hasActChange` → `buildActAmendmentDecision` |
| Пречистен текст на Актот (ДООЕЛ=Изјава / ДОО=Договор) | секогаш кога има претходната одлука | `buildConsolidatedActDOOEL` / `buildConsolidatedActDOO` |
| Договор за пренос на удел (нотарска заверка) | M5 | `buildTransferContract` |
| Понуда за пренос — до Друштвото | M5 | `buildOffer` |
| Понуда за пренос — до СЕКОЈ друг постоен содружник | M5 (по содружник) | `buildOffer` за `m5.others` |
| Понуда за пренос — до стекнувачот | M5 кога стекнувачот е ново лице | `buildOffer` |
| Изјава за прифаќање (стекнувач) | M5 | `buildAcceptance` |
| Изјава за неприфаќање — од Друштвото | M5 | `buildRejection(ctx,'company')` |
| Изјава за неприфаќање — од секој содружник што не презема | M5 (по содружник) | `buildRejection(ctx, s)` |
| Одлука за пренос на удел, истапување и пристапување | M5 | `buildTransferDecision` |
| Пријава по член 200 | M5 | `buildArticle200Application` |
| Книга на удели | M5 | `buildBookOfShares` |
| Изјава по чл. 29/183/231 (ново лице) | нов управител и/или нов содружник | `buildNewPersonStatement` |
| Одлука за организирање на подружница | M8 | `buildBranchOrganizationDecision` |
| Полномошно за организирање (company-level) | M8 | `buildBranchOrgPOA` |
| Изјава за потписи (branch wording, по потписник) | M8-only | `buildBranchSignatureStatement` |
| Полномошно — работно време (company-level) | M8 | `buildBranchWorkingHoursPOA` |
| Напомена: ЗП образец | M4 или M5 со нов управител | спомнат во чл. 32 / не се генерира |

> **M8 е самостоен (self-contained):** пакет САМО со M8 ги генерира точно адвокатските 5 документи
> (одлука + полномошно + изјави за потписи + полномошно работно време) и **НЕ** генерира генеричка
> чл.32 / по-потписник изјави/полномошна / акт документи. Комбиниран со други промени, M8 додава
> само одлука + двете полномошна (работно време вклучително), а генеричката машинерија ги покрива
> потписите. [code: `m8Only` во `assembleChildren`]

> **НЕ се генерираат (само се споменуваат):** ЗП образец, ПП пријави на ЦРМ, уверенија УЈП/Царина,
> упис на вистински сопственик, ФДИ регистрација.

### 2.3 РЕДОСЛЕД НА СОСТАВУВАЊЕ [code: `assembleChildren`]
1. **M5 документи** (ако има): договор → понуди (Друштво, секој содружник, стекнувач) → прифаќање → неприфаќање (Друштво, секој содружник)
2. **Одлуки по промена** (M1, M2, M3, M4, M6, M7 — по тој редослед)
3. **M5 одлука** + **Пријава чл. 200**
4. **Одлука 252/253** + **Пречистен текст**
5. **Книга на удели** (M5)
6. **Изјава чл. 32** → **Изјави за ново лице** (нов управител, нов содружник)
7. **Изјави за потписи** (по потписник) → **Полномошна** (по потписник)

Сите документи се одделени со прелом на страница (page break).

## 3. МАТРИЦА НА ПОТПИСНИЦИ (КОЈ ШТО ПОТПИШУВА)

> Основно правило: **одлуките ги донесуваат содружниците.** ДООЕЛ = единствениот содружник;
> ДОО = СИТЕ содружници. [code: `shareholderSignatures`]

### 3.1 ПРАВИЛО ЗА НОВИ ЛИЦА (Мартин) — ВАЖНО [code: `decisionSignatures`]
**Секоја одлука** (M1–M7 и Одлуката 252/253) дополнително ја потпишува:
- **новиот управител** (M4 a/b) — улога „ИДЕН УПРАВИТЕЛ"
- **новиот содружник** (M5 пристапување) — улога „СОДРУЖНИК КОЈ ПРИСТАПУВА"
(де-дуплицирано по име — лице што е веќе содружник не се повторува.)

### 3.2 Табела по документ
| Документ | ДООЕЛ | ДОО | [code] |
|---|---|---|---|
| Одлуки M1,M2,M3,M4,M6,M7 | единствен содружник **+ ново лице (§3.1)** | сите содружници **+ ново лице (§3.1)** | `decisionSignatures` |

### 3.3 Правила за форматирање на потпис [code: `signature`, `signatoryNameLines`]
- Правно лице потпишува: „За {назив}" + нов ред „Управител {застапник}".
- Лице со повеќе својства = **ЕДНА** изјава за потписи и **ЕДНО** полномошно, со сите својства наведени. [code: `deriveSignatories` спојува по име]
- Управителот (новиот, по промена) ја потпишува Книгата на удели. [code: `actManager`]

## 4. M5 — ПРЕНОС НА УДЕЛ (детално)

[code: `deriveM5`, builders `buildOffer/Acceptance/Rejection/TransferContract/TransferDecision/Article200Application/BookOfShares`]

- **Отстапувач** = постоен содружник. Се избира од листата на содружници; ако листата е празна или лицето
  не е во неа → **рачен внес** (име + адреса + ЕМБГ/пасош). Кај ДООЕЛ единствениот содружник е авто-избран.
  [code: фронтенд `M5Current`; бекенд `deriveM5` спојува совпаднат содружник со рачните полиња]
- **Стекнувач** = постоен содружник (уделот се зголемува) ИЛИ ново лице (пристапува).
- **Обем:** целосен / делумен (со процент што се пренесува).
- **Надомест:** без / со (цена, валута, начин и рок на плаќање → чл. 3 од Договорот).
- **Истапување:** отстапувачот целосно истапува / останува со намален удел.
- Кога листата на содружници е празна, отстапувачот се користи и како тековен содружник за потписите. [code: `generateCompanyChangesDoc` seeding]
- **Основната главнина НЕ се менува** со преносот — задолжителна реченица во одлуката, договорот, пријавата и книгата.
- **Понуди:** до Друштвото + до СЕКОЈ друг постоен содружник + до стекнувачот (ако е трето лице).
- **Неприфаќање:** од Друштвото + од секој содружник што не презема.
- **Нова содружничка структура:** ако отстапувачот истапува и нема други → стекнувачот станува 100%. [code: `transfereeBecomesSole`, `structure`]
- **Странски елемент** (отстапувач или стекнувач странец): додава ФДИ регистрација + уверенија УЈП/Царина во чл. 32 и полномошното. [code: `m5Foreign`]

### Каскадни предупредувања (не блокираат) [code: `validateFunction`]
- Отстапувачот е и управител и истапува → додај M4.
- ДООЕЛ што добива втор содружник → треба да премине во ДОО.
- ДОО што останува со еден содружник → називот мора да се смени во ДООЕЛ (додај M1).

## 8. ВАЛИДАЦИИ И ИНВАРИЈАНТИ
- ЕМБГ = 13 цифри; ЕДБ = 13 цифри; ЕМБС = 7 цифри.
- Странско физичко лице → пасош + државјанство; домашно → ЕМБГ.
- ДООЕЛ = точно 1 содружник; ДОО = 2+.
- Сите документи во ист пакет носат ист `decisionDate`.
- Називот во странство е на латиница (единствен исклучок од кирилица).
- Валидацијата е **предупредувачка** (не блокира генерирање). [code: `validateFunction` враќа `isValid:true`]

## 10. UX / ФОРМА (фронтенд)
[code: `CompanyChangesPage.js`, `companyChanges.js` config]
- 5 чекори: (1) избор на промени + upload, (2) друштво/содружници/управители, (3) **полиња по промена**, (4) агент/општо, (5) преглед.
- **Чекор 3 = под-визард:** секоја промена е посебен под-чекор (chips + Претходна/Следна промена).

  - **Десно** = нови податоци (празни, корисникот ги внесува).
- Нема „live preview" на документот за овој тип (`disableLivePreview: true`) — наместо тоа е приказ Тековно → Ново.
- Placeholder примери се **измислени** имиња (не реални од обрасците).
